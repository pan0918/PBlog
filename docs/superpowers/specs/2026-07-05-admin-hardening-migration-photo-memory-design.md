# 后台安全、迁移一致性与照片管理内存优化设计

## 目标

在不降低后台操作流畅性的前提下完成四项改进：

1. 消除 JWT 降级密钥并为登录接口增加可跨实例生效的限流。
2. 让内容迁移脚本可安全重复执行，同时保持现有文章 URL。
3. 保证文章与标签写入的原子性，并让内容元数据变更正确刷新静态缓存。
4. 显著降低后台相册“照片管理”页面加载手机原图时的浏览器内存占用。

## 范围

本次修改覆盖后台认证、数据库初始化与迁移、文章及标签写入、缓存刷新、文章浏览量路由和照片管理页面。不会引入新的外部存储、队列、Redis、图片 CDN 或虚拟列表，不会改变公开照片墙的交互设计。

## 认证与登录限流

### JWT

- `JWT_SECRET` 必须存在且达到最低长度；缺失或过短时，应用在加载认证模块时立即失败。
- 中间件和服务端认证函数复用同一套密钥读取及校验逻辑，避免两处规则漂移。
- JWT 校验要求有效的 `sub` 和 `username` 字段。
- Cookie 继续使用 `HttpOnly`、生产环境 `Secure` 和 `SameSite=Lax`。

### 登录限流

- 使用 Turso 表持久化登录失败状态，确保 Serverless 多实例和实例重启后仍然有效。
- 限流键由规范化用户名与客户端 IP 计算 SHA-256，不保存原始 IP。
- 同一限流键在 15 分钟内最多允许 5 次失败；达到阈值后返回 `429` 和 `Retry-After`。
- 登录成功后清除对应失败记录。
- 不存在的用户名和错误密码返回相同响应，避免用户名枚举。
- 限流模块保持独立，路由只负责提取请求信息和组织响应。

## 幂等迁移与 URL 兼容

### 文章 URL

- Markdown 相对路径去除 `.md` 后直接作为 slug，保留目录分隔符。例如 `Research/FluxMem.md` 继续映射为 `Research/FluxMem`。
- 文章详情页继续使用可选 catch-all 路由。
- 浏览量接口改为 catch-all slug 路由，使包含 `/` 的 slug 可以计数。
- 后台修改 slug 时同时刷新旧路径和新路径。

### 幂等规则

- 分类、标签、文章、项目、友链和相册继续使用其唯一业务字段判断是否已迁移。
- 说说在写入前按发布时间和正文查询已有记录。
- 留言在写入前按来源 ID；缺少 ID 时按创建时间、作者和正文查询已有记录。
- 照片在写入前按相册 ID 和图片 URL 查询已有记录。
- 迁移错误不再被空 `catch` 吞掉；脚本报告失败并以非零状态结束，避免部分成功被误认为完整成功。
- 幂等逻辑兼容此前已用随机 UUID 写入的数据，不依赖新生成的确定性 UUID 才能去重。

## 事务与缓存刷新

### 文章和标签

- 创建文章及其标签关联在同一写事务中完成。
- 更新文章及替换标签关联在同一写事务中完成。
- 任一标签不存在或写入失败时整体回滚，不留下已创建文章、空标签集或部分标签集。
- 数据库函数返回更新前后的 slug，供路由刷新缓存。

### 缓存

- 文章创建、更新和删除刷新首页、时间线、关于页及对应文章路径。
- slug 改名同时刷新旧文章路径和新文章路径。
- 分类和标签修改前查询受影响文章 slug；成功后刷新列表页和每个受影响文章页面。
- 说说、项目、友链、留言、相册与照片沿用现有定向刷新策略。

## 照片管理内存优化

单个相册通常为个位数照片，最大约 21 张，因此不引入分页或虚拟列表，避免打断连续编辑。

### 图片加载

- 卡片使用 `next/image`，优先加载 `thumbnail_url`，缺失时使用 `image_url` 并交给 Next 图片优化器生成适合网格尺寸的资源。
- 网格通过 `sizes` 限定实际请求宽度；首屏少量图片优先加载，其余图片懒加载并异步解码。
- 卡片保持固定宽高比，图片加载期间不会引发布局跳动。
- 非视口卡片使用 `content-visibility: auto` 和合理的固有尺寸提示，减少布局、绘制和合成成本。

### 状态更新

- 初次进入页面仍并行获取相册信息和照片列表。
- 添加成功后只把返回的新照片加入本地状态。
- 编辑成功后只替换对应照片对象。
- 删除成功后只从本地状态移除对应照片。
- 不再在每次操作后重新获取整个相册，避免全部图片元素重建和重新解码。
- 照片卡片拆成 memo 组件，稳定回调和 props，输入变化只重渲染当前编辑卡片。

### 流畅性

- 不卸载视口内图片，不使用滚动节流或人为延迟。
- 编辑表单即时响应；网络保存期间只禁用当前保存按钮。
- 首屏图片维持快速显示，其余图片在接近视口时加载。

## 错误处理

- 登录限流数据库失败时拒绝登录并返回服务错误，不能静默降级为无限制登录。
- 事务失败返回稳定的用户提示，服务端不向客户端暴露原始数据库错误。
- 图片增删改失败时保留当前本地列表和编辑内容。
- `401` 响应由后台请求层统一跳转到登录页，避免过期页面停留在假成功状态。

## 测试与验收

### 自动化测试

- JWT 密钥缺失或过短时失败，合法密钥可签发和校验 Token。
- 登录限流在第五次失败后锁定，成功登录清除记录，响应包含 `Retry-After`。
- 迁移保留带目录的文章 slug，并包含说说、留言、照片的去重查询。
- 文章标签创建和更新使用事务；失败时不产生部分数据。
- slug 改名、分类和标签修改刷新所有受影响路径。
- 照片管理页使用 `next/image`、响应式 `sizes`、懒加载和局部状态更新，不在增删改后全量刷新。
- 现有测试、TypeScript、ESLint 和生产构建通过。

### 手动验收

- 使用包含 21 张手机原图的相册连续滚动、编辑、保存和删除，交互无明显卡顿。
- 浏览器任务管理器或 DevTools 中的页面内存显著低于直接加载全部原图的版本，滚动离开后不持续增长。
- 旧地址 `/posts/Research/FluxMem`、`/posts/Research/RL_WestLake` 和 `/posts/Life/Podcast` 正常访问并记录浏览量。
- 连续重复运行迁移脚本不会增加说说、留言或照片数量。

## 非目标

- 不自动生成或上传永久缩略图。
- 不增加 Redis、Cloudflare KV 或新的第三方服务。
- 不实现相册分页、拖拽排序、批量上传或图片裁剪。
- 不在本次工作中重构与这些目标无关的前台视觉组件。
